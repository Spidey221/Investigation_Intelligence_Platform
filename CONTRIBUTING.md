# Contributing to IIP

First off, thank you for considering contributing to the Investigation Intelligence Platform! 

## Branching Strategy
- `main` - stable releases
- `develop` - active development
- `feature/*` - new features
- `bugfix/*` - bug fixes

## Committing Code
We use conventional commits:
- `feat: added new case dashboard`
- `fix: resolved evidence upload bug`
- `docs: updated readme`

## Pull Request Process
1. Fork the repo and create your branch from `main`.
2. Ensure `npm run lint` passes in both client and server.
3. Update the `CHANGELOG.md` with your changes.
4. Submit your PR and await review!
